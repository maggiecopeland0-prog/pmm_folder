## Audience Profiles

### Sales Reps [Sales]
- **Need:** Talk tracks, competitive positioning, quick-reference cards
- **Tone:** Confident, direct, action-oriented
- **Format:** Short bullets, no technical jargon, "say this" language
- **They care about:** How to pitch it, what objections they'll face, what's changed

### Customer Success Managers [CS]
- **Need:** Context on changes, customer-facing talking points, FAQ answers
- **Tone:** Warm, reassuring, solution-focused
- **Format:** Narrative paragraphs with clear headers, customer outcome framing
- **They care about:** How this affects existing customers, what questions they'll get

### Solutions Consultants [SC]
- **Need:** Technical specs, edge cases, integration details, API changes
- **Tone:** Precise, no hand-holding, can handle complexity
- **Format:** Detailed docs with code examples where relevant
- **They care about:** How it works, limitations, what's not supported yet

## Mixed-Audience Rules

When producing content for multiple audiences in one document:

1. **Label sections** with audience tags: [Sales] [CS] [SC] [All]
2. **Lead with [All]** content  -  what everyone needs to know
3. **Put [SC] technical details last**  -  Sales and CS can stop reading earlier
4. **Never assume technical knowledge** in [All] or [Sales] sections
5. **Use the same feature name everywhere**  -  don't use a friendly name for Sales and a technical name for SCs in the same doc
