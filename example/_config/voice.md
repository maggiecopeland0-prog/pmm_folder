# voice.md
*Layer 3 - Reference. Load this file when drafting any content.*

---

## Principles

- Lead with the point. Context and rationale follow.
- Plain language. If a simpler word works, use it.
- Structure for skimming. Headers, bullets, labels.
- Drafts over perfection. A usable first pass beats a polished final.

---

## What to Avoid

- Filler: "It's worth noting," "As mentioned above," "In order to"
- Hedging: "This might potentially..." - just say what it does
- Passive voice in instructions: "The button should be clicked" - say "Click the button"
- Empty superlatives: "powerful," "seamless," "cutting-edge"

---

## Samples

**Good:**
> `{feature_name}` is now live in the platform. It generates a `{feature_output}` on demand. Enter your search and run it - no setup required.

**Bad:**
> We're excited to announce that the `{feature_name}` feature has been released and is now available to customers in the interface, providing AI-powered `{feature_output}` of their data.
